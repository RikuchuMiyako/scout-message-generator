# scout-message-generator

ダイレクトリクルーティング向け **スカウトメール自動生成ツール**。

求人票・求職者情報・検索軸を入力すると、**企業風土と採用担当者のパーソナリティ**を
反映した、その求職者だけに宛てたスカウト文を Claude が生成します。単なるテンプレート
文章ではなく、「なぜあなたに送ったのか」が伝わる文面を目指しています。

## 特長

- **企業プロファイル方式**: 企業風土・担当者の性格・文体指針を一度作成・保存すれば、
  同一企業の複数スカウトで再利用できます（コーポレートサイト / エンゲージ /
  採用ピッチ資料 / LINE WORKS トーク履歴 などの要点を手動で集約）。
- **新卒 / 中途の書き分け**、**文字数指定（100文字単位）**、**検索軸**（年齢帯・
  経験職種・居住地）への対応。
- **求人票・求職者情報は PDF 読み取り or コピペ**の両対応。
- **プロンプトキャッシュ**により、同一企業での連続生成時のコスト・レイテンシを削減。

## セットアップ

```bash
pip install -r requirements.txt
cp .env.example .env   # ANTHROPIC_API_KEY を設定
```

API キーは環境変数 `ANTHROPIC_API_KEY`、Streamlit secrets、またはアプリ内サイドバーの
いずれでも設定できます。

## 起動

```bash
export ANTHROPIC_API_KEY=sk-ant-...
streamlit run app.py
```

ブラウザで表示される画面で、以下を行います。

1. サイドバーで **企業プロファイル** を選択（または新規作成して保存）。
2. **区分（新卒/中途）・目標文字数・検索軸** を指定。
3. **求人票** と **求職者情報** を入力（PDF または貼り付け）。
4. **「スカウト文を生成」** をクリック。

## 構成

```
app.py                 Streamlit UI
src/
  models.py            入力・企業プロファイルのデータモデル
  prompt.py            システムプロンプトとプロンプト組み立て
  generator.py         Claude API 呼び出し（ストリーミング + プロンプトキャッシュ）
  profiles.py          企業プロファイルの読み書き（JSON）
  pdf_utils.py         PDF テキスト抽出
profiles/              保存済み企業プロファイル（*.json）
tests/                 プロンプト組み立て等のテスト（API 不要）
```

## プロンプトキャッシュ設計

`system` を「共通プロンプト」と「企業プロファイル」の2ブロックに分割し、それぞれに
`cache_control` を設定しています。揮発的な入力（求人票・求職者情報・検索軸）は
ユーザーメッセージ側に置くことで、

- 同一企業の別求職者 → 共通プロンプト + 企業プロファイルがキャッシュヒット
- 企業を切り替え → 共通プロンプトはヒットし、プロファイル以降のみ再作成

となり、キャッシュ効率を最大化します。生成画面下部に毎回のトークン使用量
（キャッシュ読込/作成を含む）を表示します。

## 企業プロファイルの作り方（補足）

`profiles/sample_company.json` がフォーマットの例です。以下の情報源から要点を抽出し、
各項目（企業風土 / 担当者の性格・語り口 / 文体指針 / 訴求ポイント など）に集約します。

- 企業コーポレートサイト
- リクルートサイト（エンゲージ掲載）
- 採用ピッチ資料・その他採用資料
- LINE WORKS トーク履歴

> 将来的にこれらのソースからの自動取り込み（SharePoint / LINE WORKS 連携等）を
> 段階的に追加できる構成にしています。現バージョンは手動集約方式（MVP）です。
