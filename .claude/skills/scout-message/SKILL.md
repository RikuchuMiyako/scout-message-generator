# スカウトメール生成スキル

求職者一人ひとりに宛てた、企業の風土と採用担当者のパーソナリティを反映した
スカウトメール本文を作成する。生成はこのセッションを動かしている Claude 自身が行う
（外部 API を呼ばない）。

## このスキルを使う場面
- 「○○社のスカウト文を書いて」「この求職者向けのスカウトメールを作成して」など
- 既存の企業プロファイル（`profiles/<企業名>.json`）を使った継続的なスカウト作成
- 新しい企業の素材（サイト本文・採用資料・トーク履歴）からプロファイルを作る

## 実行フロー

### 1. 企業プロファイルの特定
- 対象企業を確認し、`profiles/` 配下の該当 JSON を Read する
  （例: `profiles/sample_company.json`）。
- 該当するプロファイルが無い場合は、ユーザーに素材の提供を促し
  「プロファイルの新規作成」（後述）を先に行う。

### 2. スカウト生成に必要な入力を集める
不足していればユーザーに尋ねる。
- **区分**: 新卒 / 中途
- **目標文字数**: 例「約400文字」（±10%目安）
- **求人票**: 本文の貼付、または PDF 添付（Read で読む）
- **求職者情報**: 経歴・スキル・志向など
- **検索軸（任意・補助情報）**: 年齢帯 / 経験職種 / 居住地

### 3. スカウト文を書く
`references/writing-guide.md` の指針と、特定した企業プロファイルの内容
（企業風土・担当者の性格・文体・訴求点・NG事項）に**必ず従って**執筆する。
担当者本人がその語り口で書いたように仕上げること。

### 4. 出力
- **スカウト文の本文のみ**を出力する。前置き・解説・文字数報告は含めない。
- 件名が有用なら、本文の前に「件名: ...」の1行のみ許容。

## 企業プロファイルの新規作成・更新
ユーザーが素材（コーポレート/エンゲージ本文、採用ピッチ資料 PDF/PPTX、
LINE WORKS トーク履歴など）を提供した場合、`references/profile-schema.md` に従って
内容を読み取り、`profiles/<企業名>.json` を作成・更新する。
- PDF は Read でそのまま読める。PPTX は `scripts/extract_pptx.py` で本文を抽出できる。
- 担当者の性格・語り口・文体は主に LINE WORKS トークの言葉づかいから推定する。
- 素材に無い事実（数値・制度・固有名詞）は創作しない。不明な項目は空にする。

## 重要な原則
- テンプレート的・量産型の文章は書かない。求職者の経歴・志向と求人内容の具体的な
  接点を見つけ、「なぜあなたに送ったのか」が伝わる文章にする。
- 誇張・事実にない情報の創作は禁止。
- 詳細な執筆ルールは必ず `references/writing-guide.md` を参照すること。
