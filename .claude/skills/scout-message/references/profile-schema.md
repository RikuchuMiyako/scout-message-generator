# 企業プロファイル スキーマと作成ガイド

企業プロファイルは `profiles/<企業名>.json` に1社1ファイルで保存する。同一企業の
複数スカウトで再利用する。ファイル名は企業名を英数字スネークケース等にしたもの
（例: `株式会社サンプル` → `sample_company.json`）。

## JSON スキーマ

```json
{
  "company_name": "株式会社サンプル",
  "industry": "業界・事業内容の簡潔な要約",
  "culture": "企業風土の要約",
  "recruiter_name": "採用担当者名（不明なら空文字）",
  "recruiter_personality": "担当者の性格・語り口。主に LINE WORKS トークから推定",
  "tone": "スカウト文に適した文体・トーンの指針",
  "selling_points": ["求職者への訴求ポイント（3〜6個程度）"],
  "ng_expressions": ["風土・トーンに照らして避けるべき表現（なければ空配列）"],
  "sources": { "corporate_site": "参照元URL等（任意）" },
  "notes": "スカウト作成に役立つ補足（なければ空文字）"
}
```

全フィールドを必ず出力する。値が無いものは文字列なら空文字 `""`、配列なら空配列 `[]`。

## 素材からの抽出方針
あなたは採用ブランディングのアナリストとして、与えられた素材から上記プロファイルを
組み立てる。

- `culture` / `industry` / `selling_points`: コーポレートサイト・リクルートサイト
  （エンゲージ）・採用ピッチ資料の記述から要約する。
- `recruiter_personality` / `tone`: 主に LINE WORKS のトーク履歴に表れる言葉づかい・
  温度感・一人称・絵文字の使い方などから推定する。トークが無い場合は、企業文化や
  採用メッセージのトーンから無理のない範囲で推定する。
- `ng_expressions`: 企業の価値観・トーンに照らして不適切になりそうな表現を挙げる。
  判断材料が乏しければ空配列にする。

### 厳守事項
- 素材に書かれていない事実（数値・制度・固有名詞など）を創作しないこと。
- 不明な項目は、文字列なら空文字、配列なら空配列にする。推測で埋めない。

## 素材の読み取り方
- **PDF**: Read ツールでそのまま読める。
- **PPTX**: `scripts/extract_pptx.py <ファイルパス>` でテキストを抽出する。
- **サイト本文・トーク履歴**: 貼付テキスト、または添付ファイルを Read で読む。

## 作成後
作成・更新した内容をユーザーに要約提示し、確認・修正のうえ
`profiles/<企業名>.json` に保存する。
